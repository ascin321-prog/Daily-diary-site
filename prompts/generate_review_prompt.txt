你要为个人公开日记站生成一篇“每日复盘”JSON。

【输出要求】
1. 只输出一个合法 JSON 对象。
2. 不要输出解释，不要输出 Markdown 代码块，不要输出额外前后缀。
3. 内容必须适合公开发布，不泄露隐私，不包含密码、验证码、精确住址、私人聊天原文、身份证号、手机号、银行卡等敏感信息。
4. 风格要求：真实、克制、温和、偏日记化，不空泛，不鸡汤，不营销。
5. 重点围绕以下四部分组织 body_markdown：
   - 今日完成
   - 卡点与问题
   - 今天的感受
   - 明天的一步

【字段要求】
输出 JSON 必须包含以下字段：
- title: 字符串，适合作为当天复盘标题，不要带日期前缀，直接概括内容
- date: yyyy-MM-dd
- slug: 小写英文、数字、连字符，且带日期前缀
- summary: 120 字以内摘要
- tags: 3 到 5 个中文标签数组
- body_markdown: 合法 Markdown 正文
- cover_image_path: 如果后续脚本能自动生成封面图，可先留默认值或合理 `/images/...` 路径；不要编造不存在的外部图片
- draft: 布尔值，默认 false

【body_markdown 模板】
body_markdown 必须遵循这种结构：

## 今日完成

- ...
- ...
- ...

## 卡点与问题

- ...
- ...

## 今天的感受

...

## 明天的一步

...

【额外要求】
- 如果素材不足，也要尽量写得真实具体，不要用空话凑字数。
- summary 要与正文一致，不能夸大。
- 标题不要写成 `2026-04-21 ...` 这种形式。
- slug 必须稳定、简洁、可读，例如：`2026-04-21-workflow-stabilized`
- 如果上下文中包含多个 agent 的近两日进展，要优先提炼成一篇统一复盘，而不是只盯主 agent。
- 如果没有真实可用图片能力，允许回退到默认封面或由后处理脚本生成简洁 SVG 封面。
